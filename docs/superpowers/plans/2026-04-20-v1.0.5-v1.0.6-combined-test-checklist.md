## v1.0.6 Fix: Screensaver Prevention During Active Playback (TV-only)

Addresses user-reported issue where CCwGTV Ambient Mode screensaver interrupted audio playback and returned users to the Home screen after ~30 min idle. Screen-wake lock is now held while audio is playing and released on every session-end path.

- [ ] 34. Screen stays awake during playback: Start a book playing, leave the TV alone for 12+ minutes (past the 10-min Ambient Mode threshold). Screen should stay on; Ambient Mode should NOT engage; audio should continue uninterrupted.
- [ ] 35. Pause releases the wake lock: Pause playback. Screen should dim normally after the idle timeout and Ambient Mode should engage as expected (standard Android TV behavior).
- [ ] 36. Close player releases the wake lock: Resume playback, then close the player (menu > Close, or TV Back / fullscreen-collapse on TV). With no playback active, Ambient Mode should engage normally after idle.
- [ ] 37. Playback failure / force-kill: Start playback, then force-stop the app (Settings > Apps > Audiobookshelf > Force Stop, or kill via adb). No orphaned wake lock should remain — verify Ambient Mode still engages after idle on the next app launch.
- [ ] 38. Sanity on phone/tablet: Install the same APK on a phone, start and stop audio playback. Screen should dim/sleep normally during playback — the new keep-awake logic must be fully TV-gated and have zero effect on mobile.
